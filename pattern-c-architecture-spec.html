Overview

What this PoC actually does

The PoC ingests a GitHub Pull Request webhook, opens a durable Temporal workflow per PR, spawns a per-workflow Docker sandbox container, and runs the Claude Agent SDK inside that sandbox. The agent reads the PR, edits files, runs ruff + pytest, commits and pushes the fix back to the branch, and posts a rolling status comment — all under hard isolation from the Worker host's credentials.

The architecture follows the Pattern-C "Temporal × Sandbox" reference architecture from the Agents Deployment at TheFork ADR. The defining rule: the Worker is a control plane, not the agent host. LLM-generated code never executes in the Worker process. Credentials never enter the sandbox env (one documented concession aside).

System schema

Components · Trusted zone

What lives on the Worker host

The trusted zone hosts every component that holds a Vault-loaded credential or sits on the durable control plane. Everything here is operated under the standard mTLS + Datadog + runbook regime.

GitHub (webhook)

External event source. Posts to POST /webhook on every PR open/synchronize. Includes X-Hub-Signature-256 and X-GitHub-Delivery headers.

Gateway (FastAPI · HMAC)

Three jobs: verifies HMAC against GITHUB_WEBHOOK_SECRET; checks RepoAllowlist.from_env() (fail-closed); projects the payload into a GitHubEvent and calls signal_with_start on Temporal.

Temporal Cloud

Managed durable-execution engine. Owns workflow state, event history, retry policy, Signals, Updates, deterministic replay. Worker Versioning pins each in-flight workflow to its start-time build_id.

Worker (control plane)

Python process running PRAutofixWorkflow + activities (provision_sandbox, run_agent_iteration, teardown_sandbox, post_status…). Never runs claude_agent_sdk.query() in-process. Dispatches into the sandbox via docker exec, observes the JSON-lines stream, heartbeats every 30 s.

credential-proxy (FastAPI)

Custodian of Vault credentials. Exposes /__token/{github|anthropic} (short-lived token fetch), /__forward (HITL gate that issues Workflow Updates), /__inject_test (unit-test surface), /healthz. Sits on both sandbox-net (internal) and default (outbound).

egress-proxy (tinyproxy)

HTTPS forward-proxy. The CLI claude reaches it via HTTPS_PROXY=http://egress-proxy:8888. Tinyproxy 1.11+ with FilterDefaultDeny Yes + FilterExtended Yes enforces an FQDN allowlist (api.anthropic.com, *.github.com, pypi.org, …).

Phoenix (OpenInference UI)

Self-hosted Arize Phoenix container. Accepts OTLP HTTP on :6006/v1/traces and renders agent traces (claude_code.interaction → claude_code.llm_request / claude_code.tool). UI at http://localhost:6006, zero credentials.

api.anthropic.com

Anthropic Messages API. Reachable only via the egress-proxy CONNECT tunnel — sandbox-net has internal: true (no NAT).

api.github.com

GitHub REST API. Used by github-mcp-server (subprocess of the CLI claude) for PR diff, comment fetch, check runs. Same tunnel as Anthropic.

Components · Untrusted zone

What lives inside the sandbox container

The untrusted zone is a per-workflow Docker sibling container spawned by the Worker via the host docker.sock. Hardening at container creation: cap_drop=ALL, no-new-privileges:true, pids/cpu/mem limits, NetworkPolicy via sandbox-net.

agent_runner.main

In-sandbox Python entrypoint. Three steps per iteration: setup_otel() to wire Phoenix; build_options() which calls httpx.get(/__token/github) to fetch the PAT; async for msg in query(...) serializing each AssistantMessage/ResultMessage as one JSON line on stdout.

claude_agent_sdk.query()

Python SDK that wraps the CLI subprocess and exposes an async iterator. Forwards ClaudeAgentOptions.env (including the OTel beta flags) to the child process.

claude CLI (Node) v2.1.150

The actual engine. Subprocess Node.js spawned by query(). Runs the plan→act→observe loop, calls api.anthropic.com, executes tools, applies permissions, loads plugins. 2.1.x ships the OTel traces beta — 2.0.65 did not.

/plugins (RO mount)

Read-only Claude Code plugin bundles. tf-guardrails = path scoping + categorical deny of Bash/WebFetch. tf-mitigations = secret-scan PreToolUse hook + signed-commit-trailer verifier.

/work (volumes_from)

Per-workflow working directory. Worker's prepare_workdir clones the repo into /tmp/autofix-{wf-id}/repo on the host; sandbox inherits the path via Docker's volumes_from. The same on-disk tree is visible both to the CLI Read/Edit and to Worker-side git push.

Trust model

The seven hard rules, mapped

RuleStatementWhere enforced
1Credentials never enter the sandboxcredential-proxy + short-lived /__token. One concession: ANTHROPIC_API_KEY is shipped in sandbox env until mitmproxy MitM lands (Open Q#1). concession
2Activity host code is provision + dispatch + observe + teardown — not the agent itselfsrc/activities/agent_iteration.py: dispatch_into_sandbox via docker exec. No in-process query().
3Sandbox lifecycle tied to workflow lifecycleprovision_sandbox at workflow start, teardown_sandbox in the finally cleanup path. GC activity reaps orphans.
4MCP servers sit outside the sandboxGitHub MCP runs as a child of the CLI; credentials sourced from the proxy, not from sandbox env.
5OTel exporter outside the sandboxPhoenix runs on the Trusted side; sandbox only emits OTLP HTTP. live-verified
6Idempotency keys for tool side-effectstool_call_key(workflow_id, iteration, tool_use_id) stamped as Autofix-Idempotency: trailer on commits.
7HITL approval gate position depends on layerIn-sandbox can_use_tool is fast-path; durable gate lives on credential-proxy /__forward which issues Workflow.execute_update.
End-to-end live-verified. Phoenix UI shows three spans after a single sandbox run: claude_code.interaction (root agent loop turn) + two claude_code.llm_request children. Pattern-C is functional in the PoC.
Open Question #1 — Anthropic API key. The current proxy stack cannot MitM HTTPS to inject credentials at the boundary, so ANTHROPIC_API_KEY is shipped into the sandbox env as a deliberate PoC concession. Production target: a mitmproxy-based credential-injector with a CA cert mounted into the sandbox.

How to reproduce the live run

cp .env.example .env  # fill ANTHROPIC_API_KEY, GITHUB_TOKEN, GITHUB_WEBHOOK_SECRET
docker compose up --build -d
# Phoenix UI → http://localhost:6006
# Temporal UI → http://localhost:8233
# Gateway     → http://localhost:8000/webhook

Smoke test from outside (manual sandbox spawn) — confirms the agent runs inside the container and emits traces:

docker run -d --name test-sbx --network agent-temporal_sandbox-net \
  -e SANDBOX_EGRESS_PROXY_URL=http://egress-proxy:8888 \
  -e HTTPS_PROXY=http://egress-proxy:8888 \
  -e CREDENTIAL_PROXY_URL=http://credential-proxy:8443 \
  -e NO_PROXY=localhost,127.0.0.1,egress-proxy,credential-proxy,phoenix \
  -e ANTHROPIC_API_KEY="$ANTHROPIC_API_KEY" \
  -e PHOENIX_COLLECTOR_ENDPOINT=http://phoenix:6006/v1/traces \
  -e OTEL_EXPORTER_OTLP_ENDPOINT=http://phoenix:6006 \
  agent-sandbox:latest sleep 600

echo 'Reply OK' | docker exec -i test-sbx python -m src.agent_runner.main

Open http://localhost:6006 — the trace timeline lists claude_code.interaction + nested claude_code.llm_request for the run.
